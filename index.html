What Is Software Architecture?


Software architecture defines the structure of a system

Why Architecture Matters


  • Determines how components interact
  • Influences scalability

Architectural Patterns


Architectural patterns provide reusable design solutions


Pipe-filter is one such pattern...

...designed specifically for systems that process data through multiple stages

What Is Pipe-Filter Architecture?


An architectural style for data processing systems


Data flows through a sequence of processing components

Filters


Filter

Processing Component



  • Receives input data
  • Processes and transforms it
  • Outputs the result

Filter Independence


Each filter performs one specific task


Filters are independent — they don't need to know how other filters work

Pipes


Filter A
→ PIPE →
Filter B

Pipes are the connections between filters

They simply pass data from one filter to another

Filters — Deep Dive


  • Independent processing components
  • Perform transformations on data
  • Each focuses on a single responsibility
  • Can be tested in isolation

Pipes — Deep Dive


  • Connect filters together
  • Carry output from one filter to the input of the next
  • Define the data flow direction

Data Stream — Deep Dive


  • Information flowing through the pipeline
  • Moves from input source to final output
  • Transformed at each stage

Part Two

WHY

Benefits and Use Cases

Why Use Pipe-Filter?


Several compelling reasons...

Modularity


Each filter performs one task


Makes the system easier to understand

Reduces cognitive load

Reusability


A filter designed for one pipeline...


...can often be reused in another pipeline


Write once, use many times

Maintainability


Changes to one filter...


...typically don't affect the rest of the system


Isolated changes = fewer bugs

Flexibility


  • Filters can be rearranged
  • Filters can be replaced
  • New filters can be added

Parallel Processing


Filters can potentially run in parallel


Improves performance on multi-core systems

Evaluation

Advantages & Limitations

Advantages


  • Modular design
  • Reusable filters
  • Easier debugging
  • Flexible pipelines
  • Supports parallel execution

Limitations


  • Data format must remain compatible between filters
  • May introduce overhead from data passing
  • Not ideal for interactive systems

When to Use Pipe-Filter


✓ Data processing applications

✓ Batch processing systems

✓ Stream processing

✓ When stages are independent

When to Avoid


✗ Highly interactive systems

✗ When filters need shared state

✗ Real-time systems with strict latency
